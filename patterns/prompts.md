# Mythic Infrastructure Pattern Prompts

## Pattern Extraction

```text
Read this source material as infrastructure architecture, not as metaphor. Identify the system roles, boundary conditions, flows, failure modes, governance rules, maintenance obligations, and design primitives encoded in the mythic language.
```

## Cooling And Water

```text
Translate the water, archive, memory, heat, and basin imagery into concrete data center cooling governance questions. Focus on public trust, thermal consequence, watershed risk, monitoring, and community consent.
```

## Mycelial Infrastructure

```text
Use the mycelial and micro-center patterns to design a rural edge-compute stewardship model. Specify node roles, maintenance responsibilities, local training, mobile units, funding durability, and failure modes.
```

## Hidden Grid

```text
Use the Pythia and underlayer-grid pattern to build a checklist for hidden infrastructure dependencies in data center proposals: transmission, interconnection, fiber redundancy, water, incentives, local capacity, and ratepayer exposure.
```

## Consent Gates

```text
Turn the door, gate, consent, threshold, routing, fallback, and trust patterns into a staged review process for AI infrastructure proposals. Include what must be true before a proposal advances.
```

## Coherence Review

```text
Evaluate whether a data center or AI infrastructure proposal is coherent across power, water, fiber, incentives, workforce, community benefit, governance, and long-term maintenance. Identify the first contradiction that appears under pressure.
```
